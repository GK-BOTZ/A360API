Welcome to A360 API

A360 API is a powerful, developer-friendly API toolkit designed to streamline a wide range of tasks. From downloading media from platforms like Instagram, YouTube, and Pinterest to translating text, scraping coupons, and accessing Binance P2P trading data, A360 API offers a robust suite of endpoints powered by FastAPI and Python.

Crafted by Abir Arafat Chawdhury (@ISmartCoder), this API is built for performance, reliability, and ease of use. Whether you're a developer, bot creator, or hobbyist, A360 API provides the tools you need to supercharge your projects.

Key Features:

  • 15+ powerful endpoints for diverse functionalities.
  • Real-time data fetching from trusted external APIs.
  • Comprehensive error handling and logging.
  • Open-source and community-driven development.

/ccgen

Description: Generates credit card numbers based on a provided BIN (Bank Identification Number). Supports AMEX and non-AMEX cards with Luhn algorithm validation.

Method: GET

Endpoint: /ccgen?bin={bin}&month={month}&year={year}&cvv={cvv}&amount={amount}

Parameters:

  • bin (required, string): Bank Identification Number (6-15 digits or with 'x').
  • month (optional, string): Card expiry month (e.g., "01").
  • year (optional, string): Card expiry year (e.g., "2025").
  • cvv (optional, string): Card CVV (3 or 4 digits).
  • amount (optional, integer): Number of cards to generate (1-2000, default 10).

Example Request:

curl -X GET "http://104.248.148.11:8000/ccgen?bin=515462&amount=2"

Example Response:


{
  "status": "success",
  "bin": "515462",
  "amount": 2,
  "cards": [
    "5154621234567890|01|2028|123",
    "5154629876543210|06|2027|456"

  "Country": "United States",
  "BIN Info": "Mastercard - Credit",
  "api_owner": "@ISmartCoder",
  "api_updates": "t.me/TheSmartDev"
}

/bindb/bin

Description: Retrieves BIN information or searches BINs by country or bank using the SmartBinDB library. Supports single BIN lookup or bulk searches with optional amount.

Method: GET

Endpoint: /bindb/bin?num={bin}&country={code}&bank={bank}&amount={amount}

Parameters: (At least one of num, country, or bank is required)

  • num (optional, string): BIN number for single lookup.
  • country (optional, string): ISO 3166-1 alpha-2 country code (e.g., "BD").
  • bank (optional, string): Bank name (e.g., "Chase").
  • amount (optional, integer): Number of BINs to return for country/bank searches (default 1000).

Example Request (Single BIN):

curl -X GET "http://104.248.148.11:8000/bindb/bin?num=515462"

/p2p

Description: Fetches Binance P2P trading data, including seller details, prices, and payment methods for specified assets and fiat currencies.

Method: GET

Endpoint: /p2p?asset={asset}&pay_type={fiat}&pay_method={method}&trade_type={type}&limit={limit}&sort_by={sort}&order={order}&min_completion_rate={rate}&min_orders={orders}&online_only={bool}

Parameters:

  • asset (optional, string): Cryptocurrency (e.g., "USDT"). Default: "USDT". Supported: USDT, BTC, ETH, BNB, BUSD, ADA, DOT, MATIC, SHIB, DOGE.
  • pay_type (optional, string): Fiat currency (e.g., "BDT"). Default: "BDT". See /p2p/currencies for full list.
  • pay_method (optional, string): Payment method (e.g., "BKASH"). Default: "ALL". See /p2p/methods for options.
  • trade_type (optional, string): Trade direction ("BUY" or "SELL"). Default: "SELL".
  • limit (optional, integer): Number of results (1-1000). Default: 100.
  • sort_by (optional, string): Sort by ("price", "completion_rate", "available_amount", "monthly_orders"). Default: "price".
  • order (optional, string): Sort order ("asc" or "desc"). Default: "asc".
  • min_completion_rate (optional, float): Minimum seller completion rate (0-100).
  • min_orders (optional, integer): Minimum seller monthly orders.
  • online_only (optional, boolean): Show only online merchants (true/false). Default: false.

Example Request:

curl -X GET "http://104.248.148.11:8000/p2p?asset=USDT&pay_type=BDT&pay_method=BKASH&trade_type=SELL&limit=2"
